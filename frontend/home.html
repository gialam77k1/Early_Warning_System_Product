Workflow

How information moves through the system.

1. Student registers

Public registration creates a student account with the default student role.

2. Admin organizes

Admins review users, assign classes, manage teachers, and maintain official records.

3. Teacher updates scores

Teachers enter or revise grades for their own classes only.

4. Admin approves

Score changes become official only after approval, protecting data consistency.

5. Student follows progress

Students view approved scores, current class info, and the latest prediction signals.

User Roles

Each role gets the tools it actually needs.

Admin

Oversees users, classes, approvals, dashboard reporting, and MLOps status.

Teacher

Works with assigned classes, filters students by class/risk, and submits grade updates.

Student

Tracks approved scores, class information, prediction results, and personal profile details.
